# Subagent Delegation Rules

## 위임 기준
- **직접 처리**: 단순 조회, 1~2개 파일 수정, 명령어 실행
- **서브에이전트 위임**: 멀티파일 변경, 리팩토링, 디버깅, 코드 리뷰, 보안 검토

## 위임 방식 (Claude Code 네이티브)
- 코드 탐색·조사 → `Explore` 에이전트 (읽기 전용 광역 검색)
- 구현 계획 설계 → `Plan` 에이전트
- 구현·리팩터·디버깅·리뷰·테스트 등 실행 작업 → `general-purpose` 에이전트에 역할별 프롬프트로 위임
- 모델 티어는 작업 복잡도로 선택: 단순=haiku, 표준=sonnet, 복잡(설계/보안)=opus

## 원칙
- 작업 성격에 맞는 에이전트 타입 선택 — 읽기 전용 탐색은 `Explore`, 실행은 `general-purpose`
- 작성(authoring)과 검토(review)는 별도 pass로 분리 — 동일 컨텍스트에서 자기 검토 금지
- 작업 완료 전 별도 에이전트로 결과 검증 (자기 검토 금지)
